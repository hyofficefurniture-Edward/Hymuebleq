# 图册分类数量与素材来源规则

- 状态：进行中
- 最后更新：2026-07-02
- 负责人：图册专项
- 关联资料：公司独立站站群、`hymueble图册/`、图册页素材
- 关联页面：`/catalogo/`、各行业图册页、各行业图册小类页

## 核心定位

Hymueble 图册不是电商 SKU 货架，而是 B2B 项目型采购入口。

图册页的作用：

- 帮拉美西语客户按空间、用途和项目场景理解我们能做什么。
- 帮客户用参考图表达需求，进入询盘、发图、定制和项目报价沟通。
- 承担小类页 SEO，而不是把每张产品卡做成独立排名页面。

不做：

- 大量独立产品详情页。
- 价格、库存、折扣、购物车、评分。
- 为了凑数量硬塞相似产品图。
- 淘宝、Amazon 式 SKU 分类和参数堆叠。

## 分类与数量规则

- 大品类拆分数量不预设，只保留客户真实会搜索、会采购、会发询盘的小类。
- 每个小类参考图数量不固定，以“能否完整表达该空间/用途”为准，宁可少而准，不硬凑 10 张。
- 分类优先级：空间/场景 > 用途 > 项目采购语言 > 单品名称。
- `mesa`、`silla`、`banco`、`sofá` 等可重复出现在不同空间页，不强行合并；只要使用场景不同，就按不同页面语境写。
- 页内筛选标签只做浏览辅助，不单独承担 SEO。
- 只有当某个筛选标签本身有明确搜索需求、内容足够厚、图片足够匹配时，后续才考虑升级为独立页面。

## 拉美分类研究流程

每做一个新大品类前，先做分类研究，再进入代码和图片接入。

研究重点：

- 查拉美西语 Google 搜索结果、竞品菜单、采购表达和行业目录。
- 优先参考墨西哥、哥伦比亚、智利、秘鲁、中美洲等西语市场。
- 观察这些词组：`muebles para...`、`mobiliario para...`、`equipamiento para...`、`fabricante de...`、`soluciones para...`。
- 记录高频空间词、用途词和采购词，再决定分类树。
- 不直接照搬中国工厂目录、英文分类或零售电商分类。

交付物：

- 推荐分类树。
- 每个小类的搜索意图说明。
- 每个小类建议参考图数量范围。
- 每个小类适合使用的产品/场景标签。
- SEO title、description、H1/H2 的西语方向。

## 图片来源规则

图片产品来源优先使用公司已有独立站站群，视为授权素材池。

使用边界：

- 站群只作为后台素材来源，不重新做成 Hymueble 前台外链入口，避免分流询盘。
- 不在页面直接热链站群图片。
- 正式使用前必须下载到本地、筛选、裁切、转 WebP。
- 正式图片放入 `public/assets/hymueble/[品类]/...`，路径按大品类和小类整理。
- 卡片文案必须按 Hymueble 西语 B2B 语气重写，不复制站群原文。

图片审查标准：

- 图片内容、产品类型、空间场景、卡片标题和 alt 必须匹配。
- 优先使用真实公司素材和项目/展厅/产品场景图。
- 站群图片不够好时，再补高端场景图或 AI 生成图；但真实公司素材优先。
- 避免低清、廉价、白底电商感、风格不一致或与卡片文字不匹配的图片。

## SEO 规则

- SEO 主体是小类页，不是产品卡。
- 页面 title、description、H1、H2 要匹配拉美客户会搜索的西语表达。
- 产品卡增强页面相关性和转化，不做独立排名页面。
- 文案围绕项目采购：空间、用途、批量、定制、材料、报价、交付沟通。
- 每个小类页保留 canonical、OG image，并按需要加入 `ItemList` JSON-LD。
- 不写价格、库存、购物车、评分、折扣。

## 执行顺序

1. 先做拉美西语分类研究。
2. 输出分类树、参考图数量建议和素材来源建议。
3. 从公司独立站站群和本地图册中收集候选图。
4. 做“图片 - 小类 - 用途 - 文案 - alt”匹配审查。
5. 下载合格图片到本地并转 WebP。
6. 复用现有 catalog 模板、卡片样式和询盘 modal 接入页面。
7. 做构建、静态 QA、浏览器/人工检查。

## 验收标准

每个新大品类图册上线前至少检查：

- `pnpm run build`
- `pnpm qa:static`
- 图册页面浏览器或人工检查
- 页面无中文残留
- 页面无外链站群图片
- 页面无价格、库存、购物车等电商化词汇
- 图片和卡片文字匹配
- CTA 能打开现有图册/询盘 modal
- 移动端不溢出
