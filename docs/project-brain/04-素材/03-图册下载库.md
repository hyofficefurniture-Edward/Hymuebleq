## 当前策略

当前不把大 PDF 直接放到 `public/`，也不做公开直链下载。原因：

- 多数 PDF 体积较大，当前 21 份源文件合计约 754.7MB。
- 第一阶段目标是获客留资，不是让客户匿名下载全部资料。
- 正式发送需要与表单接收方案、CRM、邮箱或云盘工作流一起确定。

当前前端已经完成：

- 所有图册 CTA 带 `data-catalog-interest`。
- 图册弹窗标题会显示客户请求的具体图册或分类。
- 图册表单隐藏字段 `catalog_interest` 会记录请求来源。
- `pnpm qa:catalogs` 会验证所有页面图册兴趣值都能映射到本清单。

## 分类、数量与图片来源

- 图册分类和参考图数量遵循 `docs/project-brain/04-素材/08-图册分类数量与素材来源规则.md`。
- 后续新增行业图册前，先做拉美西语分类研究，再决定小类数量和每页参考图数量。
- 公司独立站站群可作为授权素材池，但不得在 Hymueble 前台重新增加站群导流入口。
- 页面图片不得直接热链站群，必须本地化为 WebP 后再接入。

## QA 结果

最近一次验证：

```bash
pnpm build
pnpm qa:static
pnpm qa:catalogs
```

结果：

- 21 份 PDF 源文件存在。

- 页面中 41 个唯一 `data-catalog-interest` 均能映射到 manifest。
- PDF 源文件合计约 754.7MB。
- failures: 0。
- warnings: 0。

## 后续发送流程建议

上线前建议选择其中一种：

1. 表单服务/CRM 根据 `catalog_interest` 自动发送对应 PDF 或网盘链接。
2. 表单提交后进入销售邮箱，销售按 manifest 手动发送对应 PDF。
3. 大 PDF 上传到公司云盘，表单自动返回受控下载链接。

当前推荐第 2 种作为第一阶段最稳方案：简单、可控、不需要立即开发后端，也不会让资料完全公开。
