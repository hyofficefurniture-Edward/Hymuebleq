## 当前判断

v0.47 可以作为本地老板演示预览。当前版本已经形成：

`品牌定位 -> 首页信任建立 -> Virtual Showroom -> 酒店深路径 -> 项目能力 -> 工厂与质量控制 -> 联系询盘 -> Footer 二次转化`

它适合展示三件事：

1. Hymueble 不是普通展示官网，而是面向西语 B2B 项目采购的获客型独立站。
2. 网站已经有完整采购路径：看空间、看目录、看案例、看工厂、进展厅、发询盘。
3. 第一阶段以酒店家具为深路径，办公、医疗、教育、住宅保留导航和轻量入口，后续可按同一模型扩展。

当前版本不是最终上线版。演示时不要说“已经可以正式投放”，而是说“本地演示版已验证方向、结构、转化路径和视觉基调”。上线前仍需补真实 WhatsApp、表单接收、正式域名、统计代码、更多案例细字段和图册 PDF。

## 演示前准备

- 打开本地预览：`http://localhost:4321/`
- 推荐使用桌面浏览器演示，窗口宽度保持 1280px 以上。
- 手机端可以作为加分项单独展示 Header、菜单、流程模块和联系表单，不建议全程用手机讲。
- 演示前先刷新首页，确认右下角绿色 WhatsApp 浮动图标、Header 的 `Showroom` / `Cotizar`、Footer 社媒图标都正常显示。
- 首页 Virtual Showroom 已去掉重复的左侧酒店大卡，当前以四个 Matterport 展厅图卡作为主视觉入口。
- v0.47 已完成自动走查：核心页面、图册弹窗、联系表单、移动菜单和 Footer 均通过浏览器检查。
- 不主动点击 WhatsApp 发送消息，因为当前仍是占位号码。

## 推荐开场话术

这版不是简单把公司介绍搬到网站上，而是按拉美和西语客户的项目采购路径来设计。客户进入后先看到品牌和项目能力，再通过展厅、目录、案例、工厂和表单逐步建立信任。第一阶段我们把酒店路径做深，其他行业先保证入口完整，后续按同一套结构扩展。

## 每页讲解重点

### 首页

- 强调品牌定位：`Mobiliario B2B directo de fábrica`。
- 强调页面不是堆产品，而是把客户分到 `Catálogo`、`Proyectos`、`Contacto`。
- 展示 12 步流程时，只讲“从咨询到交付的项目闭环”，不要逐项解释太久。

### Showroom

- 展示四个 Matterport 展厅入口。
- 重点讲它对海外客户的价值：减少第一次沟通成本，让客户先理解空间和产品能力。

### Hoteles

- 说明酒店是第一阶段重点行业。
- 强调酒店客户常关心空间一致性、材料确认、批量生产、质检、包装和交期。

### Catálogo hotelería

- 说明分类是按项目空间，不是按单品 SKU。
- 图册弹窗只作为演示，不承诺已经真实发送 PDF。

### Proyectos hotelería

- 讲 Hilton、Kempinski、Raffles 等案例名称时，保持“案例资料已进入网站表达，但细字段还在继续入库”的口径。
- 不主动声称未经证据库确认的年份、金额、房间数量。

### Fábrica

- 重点讲工厂直供、生产能力、质量控制、出口包装。
- 工厂页是给采购方降低风险感的页面。

### Contacto

- 表单提交后展示西语成功反馈。
- 说明真实 WhatsApp 和后端接收是上线前任务，不影响当前演示验证。

## 不建议演示时做的事

- 不点击 WhatsApp 发送真实消息。
- 不说表单已经接入真实销售或 CRM。
- 不承诺所有案例细节已经完全入库。
- 不展示其他站群入口，避免造成客户分流或内部竞争。
- 不用“临时版”“半成品”等词汇；统一说“本地演示版 / versión de vista previa local”。

## 老板可能会问的问题

| 问题 | 建议回答 |
|---|---|
| 这个站和我们已有站群有什么区别？ | 这个站面向西语 B2B 项目采购，统一承接酒店、办公、医疗、教育、住宅等路径，不把流量分散到多个旧站。 |
| 为什么先做酒店？ | 酒店资料、案例和采购场景最适合第一阶段做深，可以作为模板复制到办公、医疗、教育和住宅。 |
| 现在能上线吗？ | 技术预览和页面结构已经成型，正式透明 logo 也已替换；但正式上线前还要替换真实联系方式、表单后端、域名、统计和更多案例证据。 |
| 社媒为什么放在 footer？ | 社媒是信任补充，不抢主转化。主转化仍是 Cotizar、Showroom、表单和 WhatsApp。 |
| 会不会把客户导到其他站？ | 已移除站群导流入口，当前 Footer 不再放其他独立站链接。 |

## 演示验收标准

- 核心路线 9 个节点都能打开，不出现 404。
- 桌面端无明显横向溢出、断图、遮挡、按钮挤压。
- 图册弹窗能打开并显示成功反馈。
- 联系表单能提交并显示成功反馈。
- Header、Footer、浮动 WhatsApp 和社媒图标视觉稳定。
- 页面中不出现“待确认 / 发布前补充 / placeholder / local preview”等可见半成品口径。
