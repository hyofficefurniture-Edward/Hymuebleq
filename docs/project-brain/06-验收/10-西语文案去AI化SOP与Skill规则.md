# Hymueble 西语文案去 AI 化 SOP 与 Skill 规则

日期：2026-07-03

状态：流程沉淀版。用于后续首页、Hotel、Office、Factory、项目案例、联系页、图册弹窗等文案慢改。

## 1. 总目标

把 Hymueble 西语文案从“AI 生成感、机器翻译感、空泛品牌话术”调整为更像真实面向拉美 B2B 项目家具采购客户的工厂型供应商网站。

去 AI 化不是单纯变口语，也不是删关键词。每句改写必须同时满足：

1. Google SEO：关键词保留并尽量前置。
2. 拉美 B2B 采购语感：表达像真实询价、项目沟通、图册/WhatsApp/RFQ 场景。
3. 降低客户心理压力：让客户知道资料不完整也可以先开始。

## 2. 适用范围

适用于：

- 首页首屏和 CTA。
- Hotel Furniture 页面。
- Office Furniture 页面。
- Healthcare / Education / Residential 页面。
- About / Factory 页面。
- Projects / Cases 页面。
- Contact / Quote / RFQ 页面。
- Download / Catalog 页面与图册弹窗。
- 表单引导语、页脚信任文案、共享 CTA。

不适用于：

- 未经确认的大规模页面结构调整。
- 新增 URL 或改 IA。
- 编造项目事实、证书、产能、客户名、国家、交期、MOQ。
- 把目录页改成零售电商。

## 3. 已确认事实

以下事实已由业务确认，本 SOP 下不再反复质疑：

- 24h 响应 SLA。
- 表单真实接收端。
- 图册 PDF 发送流程。
- 10.000 / 40.000 m2 展厅口径。
- 工厂面积、员工、项目数。
- 证书适用范围。
- 付款比例。
- MOQ。
- 5-8 周交期。
- 样品收费规则。
- BBK 35 天交付。
- RMB、面积、床位、房间数是否允许公开。
- 安装、售后、物流责任边界。

但不能自行新增未确认的数字、客户、国家、证书、交期、MOQ 或服务承诺。

## 4. 七步流程

### Step 1：抓取本地/拉美同行表达方式

目标：先吸收真实市场语感，再改文案。

执行标准：

- 大范围页面改写：采样 12-25 个西语/拉美相关同行网站。
- 单页或单模块改写：采样 5-8 个相关页面。
- 市场优先覆盖墨西哥、哥伦比亚、智利、秘鲁。
- 页面类型覆盖：首页、酒店家具、办公家具、工厂/能力、项目案例、联系/RFQ、目录下载。

同行质量门槛：

- 正向参考必须是高质量同行，不能随便抓一个低质量网站就当语感来源。
- 高质量同行应具备：真实企业感、清晰 B2B 定位、正常西语表达、可信项目/品类页面、明确联系/RFQ/WhatsApp 路径、页面完成度较高。
- 低质量同行，也就是明显“野鸡”的站点，不能作为正向表达依据。
- 低质量特征包括：文案混乱、机器堆关键词、页面破碎、排版粗糙、夸张虚假承诺、全是库存图、零售购物车逻辑严重、没有可信联系方式或业务路径。
- 这类站点最多只能作为反例记录，必须标注为“负面样本 / 不可学习”。
- 如果无法判断一个同行是否高质量，宁可不作为正向样本。

记录内容：

- 常见 CTA：`Cuéntanos tu proyecto`、`Cotiza tu proyecto`、`Solicita una cotización`、`Solicitar una propuesta`、`Hablar por WhatsApp`、`Solicitar catálogo`。
- 常见动词：`fabricamos`、`diseñamos`、`equipamos`、`cotizamos`、`revisamos`、`coordinamos`、`preparamos`。
- 采购材料：`planos`、`medidas`、`lista FF&E`、`cantidades aproximadas`、`fotos de referencia`、`acabados`、`materiales`、`fecha de apertura`、`espacios a equipar`。
- 反例：`soluciones integrales`、`líder en la industria`、`comprometidos con la excelencia`、`calidad superior`、购物车、SKU、库存、折扣、单品价格。

原则：

- 只提炼表达模式，不复制同行原文。
- 同行表达不能覆盖 Hymueble 的工厂型、出口型、项目型定位。

### Step 2：深度应用 B2B 心理学，降低采购压力

拉美采购客户常见心理压力：

- 不确定现在是否已经能询价。
- 图纸、FF&E、数量、材料还不完整。
- 担心一提交表单就要被要求完整方案。
- 担心供应商不了解项目采购、物流、包装、交期、安装边界。
- 不知道提交后下一步会发生什么。

文案处理原则：

- 用 `puedes enviar` 代替强命令式 `envía`。
- 用 `si ya tienes` / `si todavía no tienes` 说明资料可有可无。
- 用 `no son obligatorios para iniciar` 降低附件门槛。
- 用 `datos iniciales`、`cantidades aproximadas` 降低精确填写压力。
- 用 `podemos revisarlos después` 保留后续沟通空间。
- 用 `te indicamos el siguiente paso` 降低不确定性。

心理学不是煽动，不写鸡血销售话术。重点是：

- 降低不确定性。
- 降低表单压力。
- 保留采购控制感。
- 用证据替代形容词。
- 让下一步明确。

### Step 3：深度遵循 Google SEO 逻辑

SEO 不只是“关键词没删”。关键词要尽量靠前。

规则：

- H1 优先保留页面主关键词：`cotización de muebles`、`muebles para hoteles`、`muebles de oficina`、`muebles a medida`。
- 首屏第一句话前 8-12 个词内尽量出现主关键词或强相关采购词。
- H2 可以自然，但不能削弱产品类别、项目场景、工厂能力、定制能力、询价路径。
- CTA 前半句优先出现 `cotización / proyecto / catálogo / WhatsApp / muebles`。
- Meta title 和 description 保留搜索意图，不写纯品牌口号。
- 不关键词堆砌，用自然采购词串联。

取舍规则：

- H1、首屏、CTA：SEO 优先。
- 说明段、表单帮助文案：自然表达和低压力优先，但不能删核心词。

### Step 4：写方案

任何实际修改前，先输出方案。

方案字段：

| 字段 | 说明 |
|---|---|
| 页面 | 具体 URL 或页面类型 |
| 位置/模块 | H1、hero、CTA、表单、页脚等 |
| 原文 | 当前文案 |
| 问题类型 | AI 味、机器翻译感、空泛品牌话术、采购信息不足、CTA 不自然、SEO 风险、心理压力过高 |
| 对转化的影响 | 为什么影响询价或信任 |
| 同行表达依据 | 从拉美同行提炼的表达模式 |
| 心理学/UX 依据 | 如何降低不确定性或表单压力 |
| 建议改写 | 西语草稿 |
| 保留关键词 | 主关键词和转化词 |
| SEO 前置检查 | 关键词是否出现在句首或前半句 |
| 是否需要业务确认 | 涉及数字、项目、证书、交期、MOQ 时标记 |
| 是否本轮执行 | 防止范围膨胀 |

推荐先做 1 个页面试改。对 Hymueble 来说，`/contacto/` 是优先试点，因为它最接近转化。

### Step 5：方案审批

未通过前，不改源码。

审批要确认：

- 页面范围。
- 模块范围。
- 是否包含共享组件。
- SEO 关键词护栏。
- 是否接受西语语气。
- 哪些句子本轮不改。

如果用户只通过一个页面，不得自动扩展到全站。

### Step 6：执行

执行原则：

- 只改已批准文件和模块。
- 不改 URL。
- 不改页面结构。
- 不改视觉设计。
- 不改表单逻辑或接收端。
- 不改未批准的共享 CTA、页脚、首页、Hotel、Office、Factory。
- 不引入新事实。
- 不清理无关代码。

Hymueble 第一批已验证的安全范围：

- `src/pages/contacto.astro`
- `src/components/ContactForm.astro`
- `src/components/CatalogLeadModal.astro`

共享 `CTA.astro`、Footer、首页、品类页、Factory、项目案例页必须另开小批次审批。

### Step 7：验收

代码修改后必须验收：

```bash
git diff --check
pnpm build
pnpm qa:static
pnpm qa:catalogs
pnpm qa:browser
```

说明：

- 没有改图册/目录相关文案时，`pnpm qa:catalogs` 可视情况跳过。
- `pnpm qa:browser` 需要 dev server；如未启动，先运行 `pnpm dev`。
- 文案变长时，必须看浏览器 QA 是否有移动端 overflow、长文本、按钮挤压。

人工验收清单：

- SEO 关键词是否保留并前置。
- 拉美采购语感是否自然。
- 是否降低客户心理压力。
- 是否回答真实采购问题。
- 是否没有编造事实。
- 是否没有零售电商口吻。
- 是否没有空泛品牌话术。
- 是否只改了批准范围。

## 5. 三重自检模板

### 5.1 SEO 自检

| 模块 | 关键词前置检查 | 判断 |
|---|---|---|
| H1 | 主关键词是否位于句首或前半句 | 通过/不通过 |
| 首屏说明 | 前 8-12 个词是否出现主词或强相关采购词 | 通过/不通过 |
| H2 | 是否保留类别、场景、工厂能力、定制能力或询价路径 | 通过/不通过 |
| CTA | 是否含 `cotización / proyecto / catálogo / WhatsApp / muebles` | 通过/不通过 |
| Meta | title/description 是否保留搜索意图 | 通过/不通过 |

### 5.2 拉美 B2B 用语自检

| 检查项 | 判断 |
|---|---|
| 是否使用 `cotizar / cotización / proyecto / catálogo / WhatsApp` | 通过/不通过 |
| 是否出现 `planos / cantidades / referencias / lista FF&E` 等采购材料 | 通过/不通过 |
| 是否避免西班牙本土或过度正式机构腔 | 通过/不通过 |
| 是否避免零售电商词 | 通过/不通过 |
| 是否避免空泛品牌词 | 通过/不通过 |

### 5.3 心理压力自检

| 压力来源 | 降低方式 | 判断 |
|---|---|---|
| 资料不完整 | `si todavía no tienes... podemos revisarlos después` | 通过/不通过 |
| 数量不准确 | `cantidades aproximadas` | 通过/不通过 |
| 表单压力 | `5 datos iniciales` | 通过/不通过 |
| 附件压力 | `no son obligatorios para iniciar` | 通过/不通过 |
| 下一步不清楚 | `te indicamos el siguiente paso` | 通过/不通过 |

## 8. 分批完成全站文案去 AI 化目标

最终目标：用多批次、小范围、可验收的方式，完成 Hymueble 全站核心西语文案去 AI 化，让每个页面都同时满足 Google SEO、拉美 B2B 采购语感和低心理压力询价表达。

执行方式不是一次性全站重写，而是按页面价值和风险分批推进：

| 批次 | 页面/模块 | 目标 | 执行门槛 |
|---|---|---|---|
| 第 1 批 | `/contacto/`、表单、图册弹窗 | 先打磨最接近转化的询价路径，验证语气是否正确 | 已完成试点后复盘 |
| 第 2 批 | 首页首屏、首页核心 CTA、页脚信任文案 | 统一第一印象和全站转化入口，保留主关键词与工厂型定位 | 需单独方案审批 |
| 第 3 批 | Hotel Furniture、Office Furniture | 优先处理搜索意图最强的两类商业页面，保留 `muebles para hoteles`、`muebles de oficina`、`fabricante/proveedor` 等主词 | 需同行研究和 SEO 表 |
| 第 4 批 | Salud、Educación、Residencial | 处理行业场景页，减少模板话术，补足采购场景、材料、数量、项目流程语义 | 需逐页试改 |
| 第 5 批 | About / Factory | 把工厂能力、证据、质控、包装、出口配合讲具体，减少品牌口号 | 需核对事实边界 |
| 第 6 批 | Projects / Cases | 强化案例可信度，区分真实项目和参考场景，避免过度承诺 | 需核对项目公开口径 |
| 第 7 批 | Catalog / Download / 共享 CTA | 统一图册、下载、WhatsApp、询价入口语气，避免零售电商感 | 需检查共享组件影响范围 |

每一批必须重复七步流程：

1. 只采样高质量拉美/西语同行。
2. 归纳可学习表达和反例。
3. 做心理压力与表单/询价摩擦分析。
4. 做 SEO 关键词前置方案。
5. 输出试改稿并审批。
6. 小范围执行。
7. 构建、QA、浏览器和三重文案自检。

完成标准：

- 全站核心页面没有明显 AI 生成腔、机器翻译感和空泛品牌话术。
- 每个页面主关键词、辅助关键词、转化关键词都保留且尽量前置。
- 页面表达回答采购客户真实问题：能不能定制、要给什么资料、如何报价、后续怎么沟通、工厂和出口配合能力是什么。
- 询价路径更低压力：客户资料不完整也知道可以先发起沟通。
- 所有改动都有方案、审批、执行记录和验收结果。

## 9. 一句话原则

先研究拉美真实表达，再用心理学降低询价压力，在 Google SEO 关键词前置护栏内写方案；方案通过后小范围执行，最后用构建、静态 QA、浏览器 QA 和三重文案自检验收。
