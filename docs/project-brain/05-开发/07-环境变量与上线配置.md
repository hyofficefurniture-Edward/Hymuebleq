## 当前原则

全站公开联系方式继续从统一配置入口读取。WhatsApp 和邮箱可以通过公开环境变量覆盖，部署平台仍建议显式配置。

当前表单接收不再依赖 `PUBLIC_HYMUEBLE_FORM_ACTION`。联系页和图册弹窗表单默认提交到 Web3Forms，`access_key` 属于公开提交标识，不是服务器密钥；真正的安全边界必须配置在 Web3Forms 后台。

## 当前表单模式

当前实现不再区分“本地演示 mode”与“上线 action mode”：

- WhatsApp 使用当前确认的公开号码，或由 `PUBLIC_HYMUEBLE_WHATSAPP_URL` 覆盖。
- 邮箱使用当前确认的公开业务邮箱，或由 `PUBLIC_HYMUEBLE_EMAIL` 覆盖。
- 表单默认提交到 Web3Forms。
- 生成后的 HTML 不包含 `data-demo-form`。

这意味着“表单能不能正式投放”不再取决于前端隐藏字段是否可见，而取决于 Web3Forms 后台是否已经补齐域名限制、验证码、频率限制、告警，以及是否完成真实测试询盘。

## Web3Forms 安全边界

当前前端会公开输出：

- `action="https://api.web3forms.com/submit"`
- `access_key`
- 业务公开邮箱与 WhatsApp 入口

因此上线前必须在 Web3Forms 后台完成：

1. 只允许正式域名提交。
2. 开启验证码或等效的人机校验。
3. 配置频率限制 / 反滥用策略。
4. 配置异常提交告警。
5. 做一条真实测试询盘，确认文本字段与附件都能收到。

## 本地配置操作卡

本机如需覆盖公开联系方式，应复制 `.env.example`：

```bash
cp .env.example .env.local
```

然后只在 `.env.local` 填写公开联系方式：

```bash
PUBLIC_HYMUEBLE_WHATSAPP_URL=https://wa.me/8615019774832?text=Hola%20Antonio%2C%20quiero%20solicitar%20una%20cotizaci%C3%B3n%20para%20un%20proyecto%20de%20mobiliario%20comercial.%20%C2%BFPodr%C3%ADas%20ayudarme%3F
PUBLIC_HYMUEBLE_EMAIL=ao@hysdfurniture.com
```

注意：

- `.env.local` 不应提交到 GitHub。
- `.env.example` 只能放公开联系方式示例，不能放任何私密配置。
- `PUBLIC_` 开头的变量会进入前端静态页面，因此这些值不能当作秘密保存；它们只能放公开联系信息。
- 如果未来改成别的表单服务且需要私密 token，不要把 token 放进前端；应使用后端、serverless function 或服务商自己的后台安全配置。

配置后重新构建并检查：

```bash
pnpm qa:prelaunch
```

## 部署平台配置操作卡

发布到 GitHub 后，在部署平台配置两项公开环境变量：

- `PUBLIC_HYMUEBLE_WHATSAPP_URL`
- `PUBLIC_HYMUEBLE_EMAIL`

部署平台配置完成后，需要重新触发一次生产构建。Web3Forms 的接收地址和公共提交标识当前由源码输出，不通过部署平台注入。

## 上线前最终验收

1. 部署平台构建成功。
2. 线上页面 WhatsApp 链接打开真实号码。
3. Footer 和隐私页显示确认后的业务邮箱。
4. 联系页和图册弹窗表单继续指向 Web3Forms，且接收端已确认能收到附件。
5. Web3Forms 后台已完成域名限制、验证码、频率限制和告警。
6. 提交一条测试询盘，确认能在表单接收端收到。
7. 测试询盘完成后，如包含真实客户资料，应在测试工具中删除或标记为测试数据。

`pnpm qa:launch` 会检查：

- 公开联系方式是否一致。
- WhatsApp 是否仍为占位号码。
- 生成 HTML 是否仍有演示表单标记。
- 生成 HTML 是否仍指向 Web3Forms。
- TikTok 是否存在，Pinterest / X 是否回流。

## 现阶段说明

当前站点已经不是“前端 demo 表单 + 上线后再切 action”的模式，而是“默认 Web3Forms 前端提交 + 后台安全约束必须补齐”的模式。

若后续要进一步降低第三方滥用面，最小改动方案是改成自有 serverless 中转，再由 serverless 持有私密服务端密钥并转发到邮件、CRM 或表单服务。
