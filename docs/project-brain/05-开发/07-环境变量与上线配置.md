# 环境变量与上线配置

- 状态：v0.84 已建立
- 最后更新：2026-06-30
- 负责人：开发专项
- 关联页面：全站 WhatsApp、Footer、隐私页、联系表单、图册弹窗表单
- 下一步：用户确认真实 WhatsApp、业务邮箱和表单接收端点后，写入本机 `.env.local` 或部署平台环境变量。

## 当前原则

真实联系方式和表单接收地址不写死在源码里。当前源码只保留本地演示 fallback，正式上线前必须通过环境变量替换。

## 环境变量

| 变量 | 用途 | 是否公开 | 当前状态 |
|---|---|---|---|
| `PUBLIC_HYMUEBLE_WHATSAPP_URL` | 全站 WhatsApp 浮标、Footer、联系页 CTA | 是，生成进静态 HTML | 待真实号码 |
| `PUBLIC_HYMUEBLE_EMAIL` | Footer、隐私页、Organization JSON-LD | 是，生成进静态 HTML | 待确认业务邮箱 |
| `PUBLIC_HYMUEBLE_FORM_ACTION` | 联系页和图册弹窗表单提交地址 | 是，生成进静态 HTML | 待选择表单服务/CRM/后端 |

示例文件：

- `.env.example`

本机真实值应放入：

- `.env.local`

`.env.local` 已在 `.gitignore` 中排除，不应提交。

## 本地演示模式

不配置环境变量时：

- WhatsApp 使用占位号码。
- 邮箱使用 `proyectos@hymueble.com` 作为演示默认值。
- 表单不提交到外部服务。
- 表单保留 `data-demo-form`，前端脚本拦截提交并显示西语成功反馈。

这适合本地老板演示，不代表正式上线可投放。

## 上线配置模式

配置三项环境变量并重新构建后：

- WhatsApp 链接替换为真实 URL。
- 邮箱替换为确认后的业务邮箱。
- 表单生成 `action="..."`。
- 表单不再带 `data-demo-form`，不会被本地演示脚本拦截。

上线前必须运行：

```bash
pnpm qa:prelaunch
```

## 本地上线配置操作卡

本地测试正式配置时，不要直接改 `.env.example`。应复制一份本机文件：

```bash
cp .env.example .env.local
```

然后只在 `.env.local` 填写真实值：

```bash
PUBLIC_HYMUEBLE_WHATSAPP_URL=https://wa.me/YOUR_REAL_NUMBER?text=Hola%20Hymueble%2C%20quiero%20cotizar%20un%20proyecto%20de%20mobiliario.
PUBLIC_HYMUEBLE_EMAIL=YOUR_CONFIRMED_BUSINESS_EMAIL
PUBLIC_HYMUEBLE_FORM_ACTION=YOUR_REAL_FORM_RECEIVER_ENDPOINT
```

注意：

- `.env.local` 不应提交到 GitHub。
- `.env.example` 只能放占位示例，不能放真实号码、真实表单端点或任何私密配置。
- `PUBLIC_` 开头的变量会进入前端静态页面，因此这些值不能当作秘密保存；它们只能放公开联系信息和公开表单接收地址。
- 如果表单服务需要私密 token，不要把 token 放在 `PUBLIC_HYMUEBLE_FORM_ACTION`，应使用后端、serverless function 或表单服务自己的安全配置。

配置后重新构建并检查：

```bash
pnpm qa:prelaunch
```

如果要模拟上线检查但还没有真实信息，只能使用明显的测试值，并且测试完成后不要把测试值提交。

## 部署平台配置操作卡

发布到 GitHub 后，在部署平台配置同样三项环境变量：

- `PUBLIC_HYMUEBLE_WHATSAPP_URL`
- `PUBLIC_HYMUEBLE_EMAIL`
- `PUBLIC_HYMUEBLE_FORM_ACTION`

部署平台配置完成后，需要重新触发一次生产构建。仅在本机 `.env.local` 配好不等于线上已经配好。

上线前最终验收：

1. 部署平台构建成功。
2. 线上页面 WhatsApp 链接打开真实号码。
3. Footer 和隐私页显示确认后的业务邮箱。
4. 联系页和图册弹窗表单不再是 demo 模式。
5. 提交一条测试询盘，确认能在表单接收端收到。
6. 测试询盘完成后，如包含真实客户资料，应在测试工具中删除或标记为测试数据。

`pnpm qa:launch` 会检查：

- 环境变量是否存在。
- WhatsApp 是否仍为占位号码。
- 生成 HTML 是否仍有演示表单标记。
- 表单 action 是否仍为示例地址。
